# Contributing

Thanks for wanting to help improve Tango Discographies. Contributions of data
corrections, new recordings, bug fixes, features, and documentation are all
welcome.

## Ways to contribute

- **Report a data error** — open a [Data Correction issue](../../issues/new?template=data_correction.yml). Best for a small number of fixes from a non-technical contributor.
- **Suggest a feature or report a bug** — use the [bug report](../../issues/new?template=bug_report.yml) or [feature request](../../issues/new?template=feature_request.yml) templates.
- **Submit code or data directly** — open a pull request (see below).

For open-ended questions, use [Discussions](../../discussions).

## Data contributions

The per-artist CSVs in `csv_files/` are the source of truth for all data. The
compiled `discographies.csv` served by the website is regenerated by CI from
these files — you never need to edit it.

**When opening a data PR:**

- Edit files only in `csv_files/` (one artist per file).
- Preserve the existing column schema — don't reorder, rename, or add columns.
- Cite a source in the PR description for factual corrections (liner notes, a
  reputable catalog, a specific recording). Unsupported changes may be asked
  for evidence before merging.
- Run `python check_data_quality.py` locally before pushing; fix anything that
  looks like a regression.
- Keep each PR focused: one artist, or one category of fix, per PR. This makes
  review tractable.

## Code contributions

- **Python:** 3.10+, standard library only. Open a discussion before adding any
  third-party dependency.
- **Frontend:** plain HTML/CSS/JS. There is no build step and no framework,
  and we'd like to keep it that way.
- **Testing:** run `python build.py` then `python -m http.server 8000` and
  verify the feature works at `http://localhost:8000`.

## Pull request checklist

Before requesting review, check:

- [ ] PR description explains the change and the motivation
- [ ] If editing data: source cited for factual changes
- [ ] If editing data: ran `python check_data_quality.py` locally with no new
      errors
- [ ] If editing code: manually tested `index.html` after `python build.py`
- [ ] Linked any related issue(s) with `Closes #N`
